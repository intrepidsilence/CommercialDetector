## Custom LLM for Commercial Detection - 2026-03-09 13:11

- **Research feasibility of a custom ML model for commercial detection** - Evaluate whether a learned model can reduce detection latency below the current 5-10s heuristic-based approach. **Problem:** The scoring engine relies on hand-tuned weights and thresholds that may plateau in accuracy/speed; a model trained on labeled transition data could learn subtler patterns and respond faster. **Files:** `commercial_detector/detection_engine.py:33-298`, `commercial_detector/signal_source.py:1-275`, `commercial_detector/transcript_analyzer.py:32-96`, `commercial_detector/config.py:45-92`. **Solution:** Key areas to evaluate: (1) Training data pipeline — the scoring engine already produces labeled state transitions with timestamped signals, which could serve as supervised training data; (2) Model architecture — compare fine-tuned tiny classifier (e.g., small MLP on signal features) vs distilled/quantized transformer vs TFLite model; (3) Inference constraints — must run on RPi 5 4GB alongside FFmpeg + Whisper (~580MB already used), inference latency must beat current 5-10s; (4) Whether the multi-signal feature vector (silence, black, loudness, scene rate, transcript scores) provides sufficient signal for supervised learning vs needing raw audio/video features.

## HDMI Capture Device Auto-Detection - 2026-03-09 13:20

- **Add device discovery for V4L2 and ALSA devices** - List available capture devices on the config page with a "Detect Devices" button. **Problem:** Capture device paths are hardcoded defaults (`/dev/video0`, `hw:1`) with no auto-detection. USB capture card numbering is unpredictable on RPi — depends on what else is plugged in, camera interface status, and ALSA ordering. Users must manually find their device paths. **Files:** `commercial_detector/config.py:22-28`, `commercial_detector/signal_source.py:147-161`, `commercial_detector/web/templates/config.html`. **Solution:** (1) Enumerate V4L2 devices via `/sys/class/video4linux/*/name` or `v4l2-ctl --list-devices`; (2) Enumerate ALSA devices via `arecord -l`; (3) Add `/api/devices` REST endpoint returning available video/audio devices with names and capabilities; (4) Add device dropdown to the Capture section of the config page, populated by the API; (5) Optionally validate selected device by running a quick FFmpeg probe.
