{# Scoring #}

Score Dynamics

Controls how the score accumulates, decays, and triggers state transitions.

Points per second the score decays toward zero when no new signals arrive. Higher values = faster return to neutral. Too high causes false exits during commercial breaks.
Score must reach this to enter COMMERCIAL state. Lower = faster entry but more false positives. A coincident silence+black scores ~13.5, so 8.0 allows single-event entry.
Score must fall to this to return to PROGRAM state. More negative = harder to exit commercial (slower but fewer false exits).
Score ceiling clamp. Prevents runaway accumulation during long ad breaks. Lower values allow faster exit when program resumes.
Score floor clamp. Prevents excessive negative accumulation during long program segments.
Ignore all signals for the first N seconds after startup. Prevents false triggers during FFmpeg initialization and stream warmup.
Once in COMMERCIAL state, stay for at least this long before allowing transition back. Prevents rapid flickering between states.
Once in PROGRAM state, stay for at least this long. Prevents re-entering commercial during brief in-program transitions.
{# Coincidence #}

Coincidence Detection

When silence and black frames occur close together, their combined score is multiplied. This is the strongest commercial boundary signal.

Maximum gap between silence and black frame events to be considered coincident. Wider windows catch more pairs but may group unrelated events.
Score multiplier applied when silence+black coincide. At 1.5x, a coincident pair scores (5+4)*1.5=13.5, crossing the default 8.0 threshold in one event.
{# Loudness #}

Loudness Analysis

Tracks audio loudness (EBU R128) and emits signals when volume shifts significantly from the running average.

LUFS deviation from baseline to trigger a loudness signal. Lower values (3) detect subtle volume changes; higher values (6+) only catch obvious jumps.
Smoothing factor for the loudness baseline (0-1). Smaller values (0.01) track long-term average; larger values (0.05) adapt faster to volume changes.
{# Scene Rate #}

Scene Rate Boost

Commercials typically have rapid scene cuts (15-30+/min) vs program content (3-10/min). A bonus score is added when the rate exceeds the threshold.

Sliding window for counting scene changes per minute. Longer windows smooth out bursts; shorter windows react faster.
Scene changes per minute to trigger the rate boost. 12/min is a good boundary between program content and fast-cut commercials.
Extra score added on each scene change when the rate exceeds the threshold. Stacks with the base scene change weight.
